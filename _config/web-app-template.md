# 웹 앱 제작 요청서 템플릿

> 이 템플릿을 사용하여 AI에게 새 웹 앱 제작을 요청할 때 일관된 품질과 서식을 유지하세요.

---

## 📝 사용자 작성 섹션 (필수 작성)

### 1. 기본 정보

**앱 ID** (영문 소문자 + 하이픈, 폴더명으로 사용)
```
예: weather-app, todo-list, calculator, habit-tracker
```

**앱 이름** (한글, 대시보드에 표시될 이름)
```
예: 날씨 앱, 할 일 목록, 전자 계산기, 습관 추적기
```

**앱 설명** (한두 줄의 간단한 설명)
```
예: 실시간 날씨 정보를 조회하고 날씨에 따른 복장 추천을 해주는 앱입니다.
```

### 2. 기능 명세

**주요 기능** (기능별로 나열)
```
- 기능 1
- 기능 2
- 기능 3
- 기능 4 (선택사항)
```

**예시:**
```
- 검색창에 도시 이름 입력으로 날씨 조회
- 현재 기온, 습도, 풍속, 강수확률 표시
- 3일 날씨 예보 제공
- 복장 추천 제시
```

### 3. 분류 정보

**태그** (검색에 사용될 태그, 쉼표로 구분)
```
예: 생산성, 유틸리티, 계산, 날씨, 목록
```

**카테고리**
```
선택지: 유틸리티, 생산성, 엔터테인먼트, 교육, 건강, 재무, 게임, 기타
```

### 4. 디자인 요구사항

**색상 선호** (선택사항)
```
예: 파란색/초록색 테마, 어두운 테마, 밝은 테마
기본값: 파란색 기반의 밝은 테마
```

**레이아웃 유형**
```
선택지: 
- 싱글 페이지 (모든 콘텐츠가 한 페이지)
- 탭형 (여러 탭으로 구분)
- 카드형 (카드 격자로 표시)
- 리스트형 (리스트 형태로 표시)
```

**기타 디자인 주의사항** (선택사항)
```
예: 모바일 반응형 필수, 다크모드 지원, 애니메이션 최소화 등
```

### 5. 특수 요구사항

**데이터 저장 여부**
```
선택지:
- 없음 (매번 초기화)
- 로컬 (브라우저 로컬스토리지 사용) - 권장 불가 (샌드박스 제한)
- 없음 (기본값)
```

**외부 API 사용**
```
예: 날씨 API, 환율 API, 유튜브 API
주의: 공개 API만 가능, CORS 이슈 고려 필요
```

**특별 기능 요청** (선택사항)
```
예: 소리 효과, 로컬라이제이션, 오프라인 지원, 인쇄 기능 등
```

### 6. 추가 참고사항

```
예: 
- 비슷한 앱이 있다면 그 앱과 어떻게 다른지
- 특정 영감을 받은 앱이나 웹사이트
- 사용자 타겟 (나이대, 기술 수준 등)
- 우선시할 기능
```

```

├── index.html          (필수: 앱 메인 파일)

```

## 📋 제작되는 파일별 템플릿

### 1. `apps/[app-id]/index.html` - 앱 메인 파일

**요구사항:**
- 완전한 HTML5 구조 (DOCTYPE, html, head, body)
- 메타 태그 (charset, viewport)
- 모든 CSS 임베드 (head의 `<style>` 태그 내)
- 모든 JavaScript 임베드 (body 끝의 `<script>` 태그 내)
- 외부 파일 참조 최소화 (아이콘 제외)
- 모바일 반응형 (viewport 메타 태그 포함)
- 접근성 고려 (시맨틱 HTML, alt 텍스트)

**기본 구조:**
```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>[앱 이름]</title>
    <style>
        /* 모든 스타일 여기에 임베드 */
    </style>
</head>
<body>
    <!-- 앱 콘텐츠 -->
    <script>
        // 모든 스크립트 여기에 임베드
    </script>
</body>
</html>
```

**체크리스트:**
- [ ] DOCTYPE 선언
- [ ] lang 속성 설정 (lang="ko")
- [ ] charset 메타 태그 (UTF-8)
- [ ] viewport 메타 태그
- [ ] title 태그
- [ ] 모든 스타일 임베드
- [ ] 모든 스크립트 임베드
- [ ] 시맨틱 HTML 요소 사용
- [ ] 모바일 반응형
- [ ] 크로스 브라우저 호환성

### 2. `apps/[app-id]/metadata.json` - 앱 정보

**필수 필드:**
```json
{
  "id": "app-id",
  "name": "앱 이름",
  "description": "앱 설명",
  "icon": "../../assets/icons/app-id.svg",
  "tags": ["태그1", "태그2"],
  "features": ["기능1", "기능2"],
  "version": "1.0.0",
  "author": "AI Generated",
  "createdAt": "2025-01-15T10:30:00Z",
  "updatedAt": "2025-01-15T10:30:00Z"
}
```

**필드 설명:**
| 필드 | 타입 | 설명 | 예시 |
|-----|------|------|------|
| id | string | 앱 고유 ID (영문 소문자 + 하이픈) | weather-app |
| name | string | 앱 표시명 (한글 권장) | 날씨 앱 |
| description | string | 한두 줄 설명 | 실시간 날씨를 조회하는 앱 |
| icon | string | 아이콘 경로 (SVG) | ../../assets/icons/weather-app.svg |
| tags | array | 분류 태그 | ["유틸리티", "날씨"] |
| features | array | 주요 기능 목록 | ["날씨 조회", "예보 제공"] |
| version | string | 시맨틱 버전 | 1.0.0, 1.1.0 |
| author | string | 제작자 | AI Generated |
| createdAt | string | 생성 시간 (ISO 8601) | 2025-01-15T10:30:00Z |
| updatedAt | string | 수정 시간 (ISO 8601) | 2025-01-15T10:30:00Z |

**검증:**
- [ ] JSON 형식 정확
- [ ] 모든 필수 필드 포함
- [ ] id와 폴더명 동일
- [ ] icon 경로 정확
- [ ] tags 배열 (최소 1개)
- [ ] features 배열 (최소 2개)
- [ ] 타임스탠프 ISO 8601 형식

### 3. `assets/icons/[app-id].svg` - 앱 아이콘 (선택사항)

**요구사항:**
- SVG 형식
- 100x100px 이상 (스케일 가능)
- 단색 또는 최대 3색
- 심플한 디자인
- 투명 배경 (viewBox로 처리)

**기본 구조:**
```svg
<svg viewBox="0 0 100 100" xmlns="http://www.w3.org/2000/svg">
  <!-- 아이콘 내용 -->
</svg>
```

## 🔄 자동 처리 프로세스

사용자가 다음 내용을 작성한 후 **AI에게 전달:**

1. 사용자 작성 섹션의 모든 필드
2. 설명 및 참고사항

**AI가 자동으로 처리:**

1. HTML 파일 생성 (완전한 기능 구현)
2. metadata.json 생성 (사용자 정보 + 자동 필드)
3. SVG 아이콘 생성 (선택사항)
4. 폴더 구조 생성

**사용자가 실행:**

1. 생성된 파일들을 `apps/[app-id]/` 폴더에 저장
2. `git add` 및 `git commit`
3. `git push` (GitHub Actions 자동 실행)
4. 대시보드에서 새 앱 확인
