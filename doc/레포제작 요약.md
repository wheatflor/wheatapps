### 2️⃣ `github-actions-workflow.yml`
**용도**: GitHub Actions 자동화 워크플로우
**위치**: `.github/workflows/generate-metadata.yml`로 저장
**기능**:
- Push 시마다 자동 실행
- `apps/` 폴더 스캔
- 모든 앱의 `metadata.json` 수집
- 폴더트리 구조 생성
- 통계 데이터 생성
- `_data/` 디렉토리에 3개 파일 자동 생성
- 변경사항 자동 커밋 및 푸시

**핵심 스크립트**:
```javascript
// Node.js 기반 메타데이터 수집
// 1. buildTreeStructure() - 폴더트리 생성
// 2. collectMetadata() - 메타데이터 수집
// 3. generateStats() - 통계 생성
```

---

### 3️⃣ `dashboard-index.html`
**용도**: 대시보드 메인 페이지

**사용 가능한 모든 기능**:
```
✓ 앱 검색 및 필터링

✓ 통계 표시
✓ 실시간 데이터 로드
```

### 4️⃣ `web-app-template.md`
**용도**: 새 웹 앱 제작 요청서 템플릿
**위치**: `_config/web-app-template.md`로 저장
**구성**:
1. **사용자 작성 섹션**
   - 기본 정보 (ID, 이름, 설명)
   - 기능 명세 (주요 기능)
   - 분류 정보 (태그, 카테고리)
   - 디자인 요구사항
   - 특수 요구사항
   - 참고사항

2. **수정 금지 섹션**
   - 메타데이터 자동 생성 정보
   - 폴더 구조 규칙

3. **제작되는 파일별 템플릿**
   - `index.html` 요구사항
   - `metadata.json` 필드 설명
   - SVG 아이콘 스펙

**사용 방법**:
```
AI에게 제작 요청 시 이 템플릿 사용
→ 사용자 작성 섹션만 작성
→ 수정 금지 섹션은 건드리지 않기
→ AI가 자동으로 파일 생성
```

**주의**: `_data/` 폴더는 추적됨 (자동 생성 파일 버전 관리)

## 🚀 전체 구현 프로세스

### Phase 1: 초기 설정 (10분)

```bash
# 1. 레포지토리 생성 및 복제
git clone https://your-repo
cd your-repo

# 2. 폴더 구조 생성
mkdir -p apps _data _config assets/{icons,styles,scripts}
mkdir -p .github/workflows

# 3. 파일 배치
# 3-1. github-actions-workflow.yml → .github/workflows/generate-metadata.yml
# 3-2. dashboard-index.html → index.html
# 3-3. gitignore-file → .gitignore
# 3-4. web-app-template.md → _config/web-app-template.md

# 4. 첫 커밋
git add .
git commit -m "Initial setup: dashboard and workflow"
git push origin main

# 5. GitHub Actions 확인
# → Actions 탭에서 "Generate Metadata" 완료 확인
# → _data/ 폴더에 3개 파일 생성 확인
```

### Phase 2: 앱 추가 (반복)

```bash
# 1. 앱 폴더 생성
mkdir -p apps/my-app

# 2. index.html 작성
# 3. metadata.json 작성
# 4. (선택) SVG 아이콘 생성

# 5. 커밋 및 푸시
git add apps/my-app/
git commit -m "Add my-app"
git push origin main

# → 자동으로 메타데이터 생성
# → 대시보드에 앱 표시됨
```

## 🔑 핵심 개념

### 1. 자동 경로 생성
```
폴더 구조          자동 생성 경로
apps/calc/ ──→ apps/calc/index.html
               (metadata.json에 "path" 필드로 저장)

대시보드에서 클릭하면 이 경로로 자동 열림
```

### 2. 메타데이터 분리
```
UI 표시 정보  (metadata.json)
├─ 이름, 설명, 아이콘
├─ 태그, 기능
└─ 버전, 작성일

경로 정보 (자동 생성)
└─ apps/[app-id]/index.html
```

### 3. 자동 동기화
```
개발자가 파일 추가/수정
    ↓
git push
    ↓
GitHub Actions 트리거
    ↓
메타데이터 자동 수집
    ↓
_data/ 파일 업데이트
    ↓
대시보드 새로고침하면 반영됨
```

## 📋 메타데이터 필드 참고

### 필수 필드
| 필드 | 타입 | 예시 |
|-----|------|------|
| `id` | string | `weather-app` |
| `name` | string | `날씨 앱` |
| `description` | string | `실시간 날씨 조회` |
| `tags` | array | `["유틸리티", "날씨"]` |
| `features` | array | `["검색", "예보"]` |

### 선택 필드
| 필드 | 타입 | 기본값 |
|-----|------|-------|
| `icon` | string | (자동 아이콘) |
| `version` | string | `1.0.0` |
| `author` | string | `AI Generated` |

### 자동 생성 필드
| 필드 | 생성 주체 | 형식 |
|-----|---------|------|
| `path` | GitHub Actions | `apps/[id]/index.html` |
| `createdAt` | 초기 작성 | ISO 8601 |
| `updatedAt` | 매 수정 시 | ISO 8601 |
