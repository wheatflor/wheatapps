요청목적: Git 레포지토리를 이용해 정적 html페이지를 호스팅할 예정이다.

해당 레포지토리는 다음과같이 동작한다.
1. 최상단에 대시보드 페이지가 있어서 내부의 모든 웹 앱을 열람 할 수 있다.
2. 열람방식은 세가지이다.
   검색, 폴더트리, 모든 앱
3. 검색은 검색창에 검색 후 엔터를 누르면 된다.
4. 폴더트리는 Git을 push할 때마다 자동으로 레포지토리의 폴더트리 구조 및 파일에대한 정보를 데이터파일로 자동생성하는 기능을 활용한다. 이 정보는 단순히 폴더트리 구조 외에도 모든 열람방식에서 특정 웹 앱으로 접속하는 링크를 제작할 때 활용한다.
5. 모든 앱은 정렬방식을 정할 수 있으며 정렬방식은 이름 오름차순 내림차순이다.
6. 대시보드에는 통계 탭이 있으며 전체 앱 수, 마지막 업데이트 날짜(년월일시분초)를 알려준다.
7. 대시보드에서 관리되는 웹 앱의 공통된 탬플릿은 알아서 잘 해주길 바란다.
8. 단, 메타데이터의 경우 주의사항이 존재한다.
   메타데이터는 어디까지나 UI에 표시하기 위한 정보이다. 웹 앱의 접속주소는 폴더트리 기반으로 push할 때 자동생성되는 구조를 만들어서 그 것만을 사용한다.
   세부 앱의 메타데이터 정보도 필요하다면 대시보드에서 사용할 수 있어야한다.(직접 사용하는게 불가능하다면 push마다 정보를 수집해 대시보드의 데이터 파일을 만드는 과정에서 같이 수집한다.)
   관리되는 웹앱의 메타데이터는 다음의 정보를 포함한다.
   이름(주소와 별개), 설명, 아이콘, 태그, 기능
9. 추후 AI를 통해 새 웹 앱을 제작할 때 일관된 품질과 서식을 유지하기 위한 
   웹 앱 제작 요청서 탬플릿을 만들어서 제공하라.
   웹 앱 제작 요청서 탬플릿은 다음과 같은 구성으로 만든다.
   사용자가 직접 작성해야되는 섹선, 수정하면 안되는 섹션, 제작되는 파일별 탬플릿


요청사항: 요청사항에 맞춰서 동작할 수 있도록 레포지토리에 업로드할 파일과 구조를말하라
